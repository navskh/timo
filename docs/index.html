기획 도구가 아니라 실행 루프입니다

계획만 쌓이고 실행이 안 되는 게 문제잖아요. TIMO는 처음부터 실행에 초점을 맞췄어요.

⚡

Run All — 자동 루프

pending 태스크를 정렬 순서대로 한 번에 실행. 실패하면 그 자리에서 멈추고 원인을 보여줘요. "조용히 스킵" 같은 거 없음.

🔄

실시간 태스크 동기화

AI가 작업하면서 호출하는 TodoWrite가 사이드바에 즉시 반영. 진행 상황을 글자가 아니라 체크박스로 봅니다.

✨

AI가 태스크를 정리

쌓인 태스크 중 끝난 것·중복된 것·이름이 모호한 것을 한 번 클릭으로 정리. 매번 청소를 사용자가 안 해도 됩니다.

🛠️

스킬 (`/plan`, `/build`, `/review`...)

자주 쓰는 프롬프트 패턴을 슬래시 커맨드로. 직접 추가·수정 가능, 파일 한 장이면 됨.

⏸️

언제든지 중단

AI가 길게 헤매면 ⏹ 한 번. SIGTERM으로 깔끔하게 멈추고 다시 시작 가능.

⚠️ 첫 실행 시 가이드 — 꼭 읽어주세요

TIMO는 아직 Apple/Microsoft 코드 서명 인증서가 없어요. 그래서 첫 실행 시 OS가 한 번 차단합니다 — 한 번만 우회하면 그 다음부턴 정상 실행돼요.

🍎 macOS
  1. .dmg 마운트 → TIMO를 응용 프로그램으로 드래그
  2. 응용 프로그램에서 TIMO 더블클릭 → "확인할 수 없습니다" 경고 뜸 → 완료 클릭
  3. 시스템 설정 → 개인정보 보호 및 보안 → 맨 아래 스크롤 → "TIMO이(가) 차단되었습니다" → 그래도 열기 클릭
  4. 비밀번호/Touch ID 확인 → 실행 완료

첫 실행 한 번만 이 절차. 다음부턴 그냥 더블클릭으로 열림.

"그래도 열기" 후에도 앱이 떴다 사라지거나 dock에 안 나타나면:

codesign --force --deep --sign - /Applications/TIMO.app

터미널에서 한 번만 실행 → 다시 우클릭 → 열기. (sidecar에 nested signing 적용)

옛날 버전(v0.4.4 이하)에서 "TIMO이(가) 손상되었기 때문에..." 경고가 뜨면:

xattr -cr /Applications/TIMO.app

v0.4.5부터는 ad-hoc signing이 들어가서 이 명령 불필요.

🪟 Windows
  1. .exe 또는 .msi 더블클릭 → SmartScreen 차단 화면
  2. "추가 정보" 클릭 → "실행" 버튼이 나타남
  3. 설치 진행
🛠️ 첫 채팅 전

TIMO는 사용자가 이미 설치한 claude · gemini · codex CLI를 호출합니다. 셋 중 적어도 하나가 PATH에 있어야 채팅이 돌아가요. 터미널에서 which claude로 확인 가능.
